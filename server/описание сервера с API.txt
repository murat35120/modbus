СЕРВЕР
0) массив объектов контроллер
1) объекты для каждого контроллера, хранят:
	- его текущее состояние, 
	- настройки, 
	- список доступных методов 
	- подписка на события контроллера
	- триггеры контроллера (создают события)
	- номер конвертера в массиве конвертеры
2) список карт с последним событием. Номер, дата/время, контроллер, считыватель
3) методы сервера
	- HTTP server: опрос сервера, настройка сервера, команда серверу 
	- websocket: все возможности
	- HTTP client: хуки в соответствии с подпиской
	- поиск и подключение к конвертеру
	- поиск контроллера по команде старт/стоп
	- сканирование контроллеров  по команде старт/стоп
	- изменение адреса контроллера и его имени
	- авторизация на сервере
	- страница админа на сервере (websocket) 
4) методы контроллера
	- звуковая индикация (пикает)
	- световая индикация (мигает)
	- открытие замка (комплект): замок, свет, звук, реакция на датчик двери
	- свободный проход/блокировка (комплект)
5) лицензирование по номерам контроллеров, привязано к экземпляру сервера.
	- сервер отправляет сообщение (json объект) со: списком номеров и моделей контроллеров, номером экземрляра сервера, версия сервера и хеш от пакета + соль.
	- лицензия содержит спискок контроллеров, номер экземрляра сервера и хеш от пакета + соль1.
	- номер экземпляра получается с сервера вместе с первой лицензией на основе start_id (случайное число). номер экземпляра и start_id имеют разную длину
