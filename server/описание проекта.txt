начинаем разработку  от апи связи
получаем имена переменных, они используются и во фронт и в бак и в базе
все имена уникальны, при необходимости, перед имененм вставляем имя блока и "_"

Старт страницы 
- определяем домен
- ищем поле abonent с доменной приставкой в localStorage
- распаковываем
- авторизуемся на сервере по ключу сессии
- строим вспомогательные объекты 
- строим страницу


страница 
строится скриптом
делим на однотипные элементы и привязываем к функции
вспомогательные для удобства написания кода
на верхнем уровне получаем полный объект данных и строим его.
При создании блоков с именами выполняем привязку блоков к объекту links.
- базовая функция создания блока. Можно передать объект (первым параметром), можно передать набор данных.
- данные  <div>: text, class, name.  Все поля не обязательные (для разных функций набор разный).
- данные передаем в dataset: name, class
- набор функций create_div, create_span, create_input, create_text_area и тд.
- функции строители находятся в объекте builder

функции связи
это генераторы. содержат список имен отправки или ожидания получения
используем фиксированные имена полей (если проблема то конвертер имен)
- вспомогательная функция пакетного поиска имен

функции интерфейса
- объект линк для поиска нужной функции по типу активного элемента
индивидуальная, много в блоке и тд
 

 
стили 
привязаны к датасет элементов
типовые цвета
типовые шрифты
типовые отступы
флекс форматирование
 
меню настроек и авторизация 
над полупрозрачным экраном 
абсолютное позиционирование
сбоку и по центру соответственно
 
настройки и справка по бокам
выплывают по наведению мыши в эти места
если не хватает места то накрывают основную информацию 
 
за экран ничего не выходит
если данных много, используем умный поиск
