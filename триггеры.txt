триггер это функция 
массив настроек хранится в контроллере (может быть set)
в него передаются настройки в виде объектов (добавляются и удаляются)

type: имя контролируемого параметра, sost: значение при переходе в которое срабатывает триггер (0, 1)
для напряжений: min: минимальное значение, max: максимальное, threshold: порог превышения значения,
calback: функция которая вызывается при срабатывании триггера 
 
на основе этой информации формируется массив шаблонов по которым срабатывает триггер.
При срабатывании триггер вызывает все подписанные на него calback функции с параметром data

ЛОГИКА РАБОТЫ ФУНКЦИИ
2 части
1)метод контроллера set_trigger 
- получает и сохраняет настройки,
- создает массив шаблонов (до 4 шт)
- создает массивы calback вызовов
2) внешняя функция trigger получает ссылку на контроллер и data
- получает из контроллера массив шаблонов  (до 4 шт)
- проверяет по шаблонам data
- если есть изменения проверяет массив значенией
- если есть совпадения по значениям выполняет вызовы по соответствующему массиву вызовов[]
