Задание
Приложение для работы с конвертерами Z-397 WEB

несколько серверов
- UDP Сервер для поиска конвертеров в режиме сервер
- TCP Сервер для продключения конвертеров в режиме клиент
- HTTP Сервер для показа страниц управления и получение команд POST
- HTTP клиент для работы с внешним сервером, используется для хранения JSON объектов задач (Будет создан позже)

В консоли приложения выводятся локальный и сетевой адрес сервера HTTP,
если компьютер подключе к нескольким сетям, то информация о сетевом адресе выводится для всех сетей.

модули, оформляются в виде отдельных файлов в папке moduls
- модуль генерации главной страницы
- модуль класса Конвертер
- модуль объекта  поиска конвертеров
- модуль  API работы с приложением
- модуль  API работы с внешним сервером  (Будет создан позже)

Папка pages для хранения HTML страниц
1) Страница Оборудование
Состоит из трух вертикальных блоков
Левый блок - список конвертеров
Центральный блок - информация о выбранном оборудовании
Правый блок - список найденых конроллеров
Левый блок разделен на три части по вертикали. В верхней находится кнопка включить/выключить UDP поиск, в средней части список найденых конвертеров в режиме сервер, в нижней список конвертеров в режиме клиент.
Пользователь , нажимет на кнопку выбранного конвертера. В центральном блоке выводится информация об этом конвертере, там же можно подключить и отключить конвертер.
После этого, связь с конвертерами восстанавливается автоматически, пока Пользователь их не отключит. Средствами CSS блоки контроллеров оформлены в соответствии с их текущим состоянием
Центральный блок разделен на две части. В верхней части поле с выпадающим списком выбора рабочей скорости  230400,  115200, 38400 (по умолчанию) и кнопки управления.

Модуль генерации главной страницы
Главная страница создаетмя после анализа содержимого папки pages.
Создается список ссылок на доступгые страницы.
Выводится название страницы и содержимое поля title
При нажатии на кнопку, открывается выбранная страница в новой вкладке браузера.
Для обновления информации о страницах, есть кнопка обновить

Модуль класса Конвертер
Содержит свойства для хранения информации о конвертере, ссылку метод записи в конвертер, ссылку на метод получения ответа конвертера

Последовательность работы с конвертерами
1) Поиск конвертеров в режиме сервер
2) Найденые конвертеры добавляются в список (Map - ключ заводской номер конвертера)
3) Подключение к конвертеру режим сервер  = создание и добавление объекта конвертер, выполняется по команде API
4) Используя протокол NVT переключается на рабочую скорость, определяется на странице Оборудование
Используется страница Оборудование

1)Конвертер в режиме клиент присылает запросы.  Подключение выполняется автоматически
2) Используя протокол NVT переключается скорость на 230400
3) Отправляется запрос информации о конвертере
4)  После успешного получения информации, конвертер добавляется в список (Map - ключ заводской номер конвертера), создается  и добавляется объект конвертер. 
5) Используя протокол NVT переключается на рабочую скорость, определяется на странице Оборудование
Операции выполняются автоматически.
6) Конвертер можно удалить, тогда подключение к нему не восстанавливается, пока Пользователь не нажмет кнопку подключить

API работы с приложением
Управляется  POST запросами
Запрос JSON строка объекта
{
  api:api,
  cmd:cmd,
  data:{
      name1:valee1,
     ......
   }
}
api - название объекта хранения команд (converter, controller, server, external_server)
cmd - название команды
data - набор отправляемых данных
Ответ содержит, данные из запроса   api,  cmd и данные ответа
{
  api:api,
  cmd:cmd,
  ansver:{
      name1:valee1,
     ......
   }
}

API работы с внешним сервером
Управляется  POST запросами
Запрос JSON строка объекта
{
  login:login
  session:session,
  api:api,
  cmd:cmd,
  data:{
      name1:valee1,
     ......
   }
}

login и session получаются после регистрации на внешнем сервере
